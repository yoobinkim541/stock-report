## 패턴 분석 — 반복된 근본 원인

1. **NaN의 비교 통과**: `NaN <= 0`은 False — 가드가 NaN을 못 막음. 수치 가드는 반드시 `np.isfinite()` 병행. (백테스트 $nan, -80% 가드)
2. **dict 키 문자열 오타는 조용히 기본값으로**: `dict.get(key, default)`는 키 오타를 감추는 안티패턴. 핵심 파라미터 조회는 키 존재 검증 또는 회귀 테스트 필수. (bull 블렌드 키)
3. **`__file__` 기반 경로는 파일 이동에 취약**: 리팩토링 시 `dirname(__file__)` 전수 grep. 가능하면 단일 `PROJECT_DIR` 상수를 한 곳에서 정의.
4. **방향 없는 수치의 배율 곱**: confidence·score 같은 무방향 수치를 방향 있는 행동(증액/감액)에 곱하면 역방향 버그. 부호/방향을 명시적으로 분리.
5. **"이상치 가드"가 실제 극단 상황을 차단**: 데이터 오류 가드(-80%)와 진짜 시장 극단을 같은 조건으로 거르면 가장 필요한 순간에 시스템이 무력화.
6. **백테스트-라이브 로직 이원화**: 별도 구현은 반드시 괴리 발생 → 헬퍼 함수 공유로 단일 소스 유지.
7. **동기 메인 루프에 무거운 작업 삽입**: 폴링 루프의 응답성은 그 안에서 실행되는 가장 느린 작업에 종속 — 주기 작업은 스레드 분리.
8. **atomic write 규칙의 부분 적용**: `holding_manager._save()`만 지키고 신규 코드(apply_snapshot, save_alerts)는 누락 — 규칙은 코드 리뷰 체크리스트로 강제 필요.
